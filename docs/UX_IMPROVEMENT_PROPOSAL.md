# ShareX 정산 시스템 — UX 개선 제안서

> 작성일: 2026-02-12
> 대상 사용자: 정산 실무 담당 이사님
> 목적: 분기별 정산 업무의 처리 시간 단축 및 오류 최소화

---

## 1. 현재 업무 플로우 분석

```
패스트캠퍼스 PDF 수신 → 수동 데이터 입력 → 엑셀 계산 → 기업별 정산서 작성 → 검토 → PDF 생성 → 이메일 발송
                         ↑ 3-5분 소요               ↑ 반복 작업                     ↑ 수작업
```

**Pain Points:**
- PDF에서 수치를 눈으로 확인하며 수동 입력 → 오입력 위험
- 13개 기업 × 각 정산서 생성 → 반복 노동
- 정산서 검토 시 기업 간 비교가 어려움
- 과거 분기와의 추세 비교 불가

---

## 2. 개선 제안 (우선순위순)

### 2-1. [P0] 정산서 상태 관리 플로우

현재는 "업로드 → 즉시 검토"의 1단계 플로우입니다. 실무에서는 아래와 같은 상태 관리가 필요합니다.

```
업로드 → 검토중 → 수정필요 → 수정완료 → 승인대기 → 최종승인 → 발송완료
```

**제안:**
- 각 기업 정산의 상태 배지 (미검토 / 검토완료 / 수정필요 / 승인)
- 기업 목록에서 상태별 필터링
- 전체 기업이 "승인" 상태일 때만 "최종 승인 및 PDF 생성" 활성화
- 승인 이력 로그 (언제 누가 승인했는지)

### 2-2. [P0] 기업별 정산서 인라인 수정

현재 웹에서 정산 데이터를 **조회만** 할 수 있습니다. 이사님이 검토 중 수치를 수정해야 하는 상황이 있을 수 있습니다.

**제안:**
- 강의별 매출액, 마케팅비 셀을 클릭하면 인라인 편집 모드 진입
- 수정 시 자동으로 공헌이익, 수익쉐어 재계산
- 수정 이력 표시 (원본 값 → 수정 값)
- "원본으로 복원" 버튼

### 2-3. [P1] 분기/연도별 대시보드

**제안 화면 구성:**

| 영역 | 내용 |
|------|------|
| 기간 선택기 | 드롭다운 (2024-Q1 ~ 현재) |
| 누적 매출 추이 | 분기별 총매출 라인 차트 |
| 기업별 정산 비중 | 파이/도넛 차트 (정산 금액 기준) |
| 강의별 매출 Top 10 | 바 차트 (해당 분기 기준) |
| 기업별 분기 추이 | 각 기업의 정산금액 분기별 비교 테이블 |
| 성장률 지표 | 전분기 대비 매출/정산 증감률 |

### 2-4. [P1] PDF 일괄 다운로드 및 이메일 발송

현재 PDF 생성 후 수동 다운로드만 가능합니다.

**제안:**
- "전체 다운로드" → ZIP 압축 파일 1개로 제공
- "개별 이메일 발송" → 기업별 담당자 이메일로 정산서 자동 발송
- "일괄 이메일 발송" → 전체 기업에 한 번에 발송
- 발송 상태 추적 (발송중 / 발송완료 / 발송실패)
- 발송 이력 로그

### 2-5. [P1] 정산서 미리보기 (Print-ready)

웹에서 보는 정산서와 PDF가 완전히 동일한 레이아웃이면, 별도 PDF 생성 없이 브라우저 인쇄로도 대응 가능합니다.

**제안:**
- `@media print` CSS로 A4 레이아웃 최적화
- 정산서 페이지에 "인쇄" 버튼 추가
- 인쇄 시 사이드바/네비게이션 자동 숨김
- PDF 다운로드와 동일한 결과 보장

### 2-6. [P2] 알림 및 리마인더

**제안:**
- 패스트캠퍼스 PDF 업로드 후 검토 미완료 시 슬랙/이메일 알림
- 분기 시작 시점에 "지난 분기 정산 마감" 리마인더
- 기업 담당자에게 "정산서 확인 요청" 이메일 자동 발송

### 2-7. [P2] 키보드 네비게이션

이사님이 정산서를 빠르게 검토할 수 있도록:

**제안:**
- `↑`/`↓` 키로 기업 목록 이동
- `Enter`로 기업 선택
- `A` 키로 승인
- `N` 키로 다음 미검토 기업 이동
- `Ctrl+S`로 데이터 저장

### 2-8. [P2] 사업자 정보 관리 화면

현재 기업 정보는 `companies.json`에서 수동 관리됩니다.

**제안:**
- 웹 UI에서 기업 정보 CRUD
- 계약 만료일 알림 (D-30, D-7)
- 기업 추가/수정 시 자동으로 companies.json 반영
- 계약 이력 관리

## 3. 마이크로 인터랙션 개선

| 현재 | 개선안 |
|------|--------|
| 파일 업로드 후 로딩 스피너만 표시 | 진행 상태 표시 (파싱중 → 계산중 → 완료) |
| 기업 선택 시 즉시 전환 | 슬라이드 트랜지션 + 스켈레톤 |
| 에러 시 빨간 텍스트만 | 토스트 알림 + 해결 방법 안내 |
| 수정 사항 저장 시 피드백 없음 | "저장되었습니다" 토스트 + 체크 애니메이션 |
| PDF 생성 완료 시 버튼만 변경 | 성공 토스트 + 파일 목록 프리뷰 |

---

## 4. 접근성 및 반응형

- 현재 데스크탑 전용 레이아웃 → 태블릿(iPad) 지원 추가 권장
- 색상 대비 비율 4.5:1 이상 확보 (WCAG AA)
- 키보드 탭 순서 최적화
- 정산 금액에 천단위 구분자 일관 적용 (완료)

---

## 5. 보안 고려사항

- 정산 데이터는 민감 정보 → HTTPS 필수 (배포 시)
- 사용자 인증 추가 (현재 미구현)
- 정산서 PDF에 워터마크 옵션 (초안/확정)
- 아카이브 데이터 접근 로그
